一、数据类型

1.1 js中的数据类型以及是如何存储的?

javaScript 一共有8中数据类型,其中基本数据类型有: null、undefined、boolean、number、string、bigint、symbol;
还有一个引用数据类型 object(function、array、Date等)
基本数据类型保存在栈中,占用空间小,大小固定
引用数据类型保存在栈和堆中，占用空间大,大小不固定,其中引用数据类型的键存放在栈中,栈中的指针指向堆,堆中储存了键所对应的值

1.2 如何判断数据类型

  • typeof
    除了null会返回object以外,其他都是对的

    function fn () {}
console.log(typeof 1);      //number
console.log(typeof '222'); // string
console.log(typeof null); // object  false
console.log(typeof undefined); // undefined
console.log(typeof NaN); // numbers
console.log(typeof [1]); // object
console.log(typeof fn ); // function
console.log(typeof {} ); // object

  • instanceof

    • 根据原型链去找类型的prototype
    • instanceof 的右边只能是一个对象
    • 主要是用来判断对象的数据类型
    • 对于基本数据类型,只有通过 new 创建的才能判断
      function fn () {}
let num = 2
let num1 = new Number(2)
console.log(num instanceof Number); // false     
console.log(num1 instanceof Number); // true  
let str = '11111'
let str1 = new String('22222')
console.log(str instanceof String); // false      
console.log(str1 instanceof String); // true  
let obj = {}
console.log(obj instanceof Object) // true
console.log(fn instanceof Function) // true
console.log({ } instanceof  Object) // true

  • constructor

    console.log((1).constructor == Number); // true     
console.log(('abc').constructor == String); // true     
console.log((undefined).constructor == undefined); // 报错  Cannot read property 'constructor' of undefined 
console.log((null).constructor == null); // 报错   Cannot read property 'constructor' of null 
console.log(([]).constructor == Array); // true 
console.log((new Date()).constructor == Date); // true 
function Fnc () {}
Fnc.prototype = new Number()
console.log(Fnc);
let fn = new Fnc(22222);
// 当改变了构造函数的原型时,对应实例的constructor也会改变
console.log((fn).constructor === Function); // false

  • Object.prototype.toSting.call() 最完美的判断数据类型的方法
    Object.prototype.toSting 返回一个对象的字符串,也就是对对象的描述 [object object]

    /**
 * 如果只传一个参数则返回参数类型
 * 如果传两个参数，则判断是否是传入类型
 * @param {String} type 需要判断的类型
 * @param {*} data 需要检测的对象
 */
function checkType (type, data) {
  if ( !data ) return  Object.prototype.toString.call(type)
  return  Object.prototype.toString.call(data) == `[object ${type}]` 
}

let arr = [1, 2, 3, 4]
console.log(checkType(arr))
console.log(checkType('Array', arr)) //true

二、作用域和作用域链

2.1 谈谈你对作用域和作用域链的了解

作用域是变量声明的时候,产生的区域,使用 var 声明的变量,一般情况下是全局变量,在全局都可以访问,而使用 let 和 const声明的变量具有块级作用域,只能在当前块级作用内访问
作用域链 通过作用域链,函数内可以访问到全局变量和父级的变量

2.2. 谈谈你对this的理解,以及各种环境下的this

  • 在浏览器里,this 指向的window对象
  • 在函数中,this 指向调用他的那个对象(箭头函数除外)
  • 在构造函数中,this 指向的是通过构造函数创建的实例
  • 箭头函数中没有this,this指向的是父级作用中的this

2.3 原型和原型链

原型: 显示原型:每一个构造函数有的prototype属性; 隐士原型:实例对象有的属性 proto ; 且实例对象的__proto__指向他构造函数的原型
原型链: 当访问实例对象的属性时,若当前的对象中没有该属性,会在其 proto 的值中去查找,也就是构造函数的prototype属性,若还没有,会继续向上查找构造函数的prototype属性的__proto__的值中是否拥有,若果没有会继续向上查找,直到返回的值是null,结束

三、 闭包

  • 定义
    • 闭包指有权访问另一个函数作用域内变量的函数,即函数中嵌套函数
  • 作用
    • 创建私有变量
    • 函数内返回一个函数
    • 函数节流和防抖
    • 延长变量的生命周期

四、事件模型

浏览器的事件模型一共有3种:

  • DOM0级模型 —没有事件流,不会传播
  • IE事件模型 —分为两个过程,事件处理阶段和事件冒泡阶段
    • 先执行绑定的事件,然后是冒泡阶段,事件从自身->父元素->body->html->document->window
  • DOM2级事件模型 —分为3个过程,事件处理阶段、事件冒泡阶段和事件捕获阶段
    • 先跟IE事件模型一样,然后执行捕获,捕获是先从window->document->html->body->父元素->自身

五、常用的event对象

  • event.preventDefault 阻止默认事件
  • event.stopPropagation 阻止冒泡

八、函数

8.1 函数的柯里化

  • 定义
    • 一种将使用多个参数转化成一些列使用一个参数的技术,返回的是一个函数的函数
  • 作用
    • 减少代码的冗余,增加代码的可读性
    • 延迟计算
    • 参数复用
  • 缺点
    • 存取arguments会占用内存
function curry(...rest){
    let _args = rest
    let _return = function (){
        _args.push(...arguments)
        return _return
    }
    _return.toString = function(){
        return _args.reduce((a,b)=>{
            return a+b
        })
    }
    return _return
}
