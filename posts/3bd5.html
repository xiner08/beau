js工具
发布日期:   2020-09-07
webpack 是一个模块打包工具,基于nodejs,只支持js和JSON模块

一、基础的一些配置(entry,mode,output等)

1.entry 入口文件的设置

  • 使用字符串 一个入口,一个出口
    entry:'./src/index.js'
  • 使用数组 可以是多个入口，但是一个出口
    entry:['./src/index.js','./src/other.js']
  • 使用对象 可以是多个入口，多个出口口 (多页面应用)
      // 3个属性名,对应了3个出口文件
entry:{ 
  main:'./src/index.js',
  other:'./src/other.js',
  others:['./src/test.js','./src/test1.js']
}

2. mode 模块打包的模式

默认值是 production 生产环境

3. output 出口文件

output:{
  //生成的出口问件 目录,绝对路径
  path:path.resolve(__dirname,'./dist'),
 //  构建的文件
  // 1.一个 出口文件 使用的是字符串的格式
  filname:'main.js'
  // 2. 出口文件 使用的是占位符的格式
  filname:'[name].js'
  // 3. 出口文件 使用的是hash(长度为6)+占位符的格式
  // 修改 一个文件,打包后所有文件的hash都改变了,整个项目的缓存文件,都将失效
  // 多个入口文件，hash值都是一样的
  filname:'[name]_[hash:6].js'
  // 4. 出口文件 使用的是chunkhash(长度为6)+占位符的格式  
  //  chunkhash 对于代码块 互不影响且不存在引用关系的代码块,改变一个文件,当前文件的hash值改变
  //另一个文件的hash值不变, 且不会再次打包
  filname:'[name]_[chunkhash:6].js'
 // 5. 出口文件 使用contenthash(长度为6)+占位符的格式  
  //  contenthash 对于一个文件，在同时引用js文件和css文件时，当js文件发生改变时，css文件不变,
  // 则 js文件的hash值改变，而css文件不改变
  filname:'[name]_[contenthash:6].js'
}

4. context 基础目录

//上下文   项目打包的相对路径 必须是绝对路径
 // process.cwd() 是当前Node.js进程执行时的文件夹目录——工作目录，保证了文件在不同
 //的目录下执行时，路径始终不变
 context: process.cwd()

5. devtool 中的sourceMap

sourceMap 打包后的文件和原文件之间的关系,如果开启,可以定位到原文件,不安全
  • mode==development 默认开启
  • mode==production 一般不建议开启,若要开启使用 cheap-module-eval-source-map
    devtool:"cheap-module-source-map"

二、module 模块转化 loader–需要安装

2.1 css-loader

  • css-loader的简单使用
    module:{
  rules:[
    {
      test:/\.css$/,
      /*
      * loader 执行顺序从后往前
      * css-loader  是将 css模块文件添加到js文件中 
      * style-loader 是将 css 出口出来 转化为以style标签的样式添加到html中
      */
      use:['style-loader','csse-loader']
    },
  ]
}
  • css-loader的模块的使用
    module:{
  rules:[
    {
      test:/\.css$/,
      /*
      * css modules 开启 
      */
      use:['style-loader',{
        loader:'css-loader',
        options:{
          modules:true
        }
      }]
    },
  ]
}

2.2 less-loader

module:{
  rules:[
    {
      test:/\.css$/,
      /*
      * css modules 开启 
      */
      use:['style-loader',{
        loader:'css-loader',
        options:{
          modules:true
        },
        'less-loader'
      }]
    },

2.3 postcss-loader 帮助css 适应css各种浏览器

1. 下载 postcss-loader 和 autoprefixer
2. 使用postcss-loader 必须是在css-loader之后
module:{
  rules:[
    {
      test:/\.css$/,
      /*
      * css modules 开启 
      */
      use:['style-loader',{
        loader:'css-loader',
        options:{
          modules:true
        },
        'postcss-loader'
      }]
    },
  ]
}
3.创建一个 postcss.config.js 文件
const autoprefixer=require('autoprefixer');
module.exports={
  plugins:[
    /**
     * autoprefixer("IE 10")
     * last 2 versions  最近的两个版本
     * >1%  全球浏览器份额大于1%
     *  也可以指定 版本号
    */
    autoprefixer({ 
      overrideBrowserslist:["last 2 versions",">1%"]
    })
  ]
}

2.4 url-loader 处理文件(包含了file-loader,且比他更强大)

module:{
  rules:[
    {
      test:/\.(png|jpe?g|gif)$/,
      /*
      * css modules 开启 
      */
      use:[{
        loader:'url-loader',
        options:{
          /**
           * ext 后缀名
           * outputPath 输出文件 加一个目录 提取图片在一个目录下
           * limit 小的文件可以将其转化为base64格式 提高加载速度 10kb以下
          */
            name: '[name]_[hash:6].[ext]', 
            outputPath:'images/',
            limit: 10*1024
        }
      }]
    },
  ]
}

三、 plugins 插件的使用

插件 作用于webpack整个打包生命周期的

3.1 clean-webpack-plugin 每次打包，清除打包后的文件

1.下载 clean-webpack-plugin 
npm install -D clean-webpack-plugin 
2.在webpack.config.js文件中引用
const {CleanWebpackPlugin}=require('clean-webpack-plugin');
3.使用
plugins:[
  new CleanWebpackPlugin({
    cleanOnceBeforeBuildPatterns: 
    [
    /*
    * 打包 不清除的文件夹和文件 前面加 ! 
    * 数组中 一定要有 **/*   //否则不生效  
    */
      "**/*", "!images","!index.html"
    ]
  }))
]

3.2 htmlWepbpackPlugin 自动生成html文件，并将打包后生成的js模块一如到html中

1.下载 htmlWepbpackPlugin
npm install --save-dev html-webpack-plugin
2. 加载模块
const HtmlWebpackPlugin = require("html-webpackplugin");
3.使用
plugins:[
new HtmlWebpackPlugin({
    //选择html模板
    /**
     *  template html的模板路径
     *  title html的标题 要在html使用 ejs 语法擦能生效
     * <title><%= htmlWebpackPlugin.options.title %></title>
     *  filename 生成的文件 
    */
    template:"./src/index.html",
    title: "心儿",
    filename:"index.html"
  })
]

3.3 mini-css-extract-plugin 抽离css

1.下载 mini-css-extract-plugin
npm install --save-dev mini-css-extract-plugin
2. 加载模块

3.使用
替换style-loader为:

 filename: "[name][chunkhash:8].css"

四、devserver的使用 本地开发服务

4.1 devserver的 简单的使用

  • 提高开发效率,更改代码后不需要重新打包
  • 打包后生成的文件不会放在dist目录下,而是放到内存中 从而提升速度
    1.下载 webpack-dev-server
npm install webpack-dev-server -D
2.使用
devserver:{
  contentBase:path.resolve(__dirname,'./dist'),
  open:true,
  post:8080
}

六、小问题总结

6.1 webpack中chunk和bundle中的关系

  • chunk 是webpack对源文件进行打包时,根据引用关系生成 chunk文件,并对其进行一些操作,不可以在浏览器运行
  • 处理好的chunk文件,最后输出成 bundle,可以直接再浏览器中运行

6.2 file-loader和url-loader的区别

  • file-loader 把打包⼊⼝中识别出的资源模块，移动到输出⽬录，并且返回⼀个地址名称
  • url-loader 包含了file-loader,但是对jpg格式的模块,会将其进行压缩,并打包到js中,大体积不合适
